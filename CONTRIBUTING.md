# 参与明徒 AI

明徒 AI 的改动应让学徒更容易被教学、更容易被纠正，或让执行证据更容易复核。新增一条很长的流程并不自动等于能力提升。

## 开始开发

```powershell
npm ci
npm run typecheck
npm test
npm run verify:plugin
```

包装工作流和 MCP 快速检查：

```powershell
npm run smoke:packaging-workflow
npm run smoke:plugin-tool-surface
```

本地 Web 产品：

```powershell
npm run dev
```

## 提交标准

- 保持 UI、API、AI 服务、执行引擎、记忆、轨迹、规则提取、知识证据和工具注册层的边界。
- 新的老师纠错默认只能生成待审核规则，不得自动启用。
- Image2 图片只提供视觉样图和形状语义，不能作为工程尺寸真值。
- 包装到 CAD 的改动必须覆盖阶段跳过、尺寸来源、自查、蒙版提交和 CAD 交接测试。
- 不公开私有思维链；使用步骤、证据、规则、置信度、验证和人工复核点组成的公开轨迹。
- 不提交 `.env`、本地数据库、运行会话、测试缓存、个人路径或未经授权的素材。

## 拉取请求说明

请写清：

1. 解决的用户问题；
2. 修改了哪些行为和边界；
3. 运行了哪些测试；
4. 截图、验证报告或结构化证据；
5. 仍需人工审核的事项。

任何自动化测试通过都不能替代老师审核、工程复核或量产验收。
