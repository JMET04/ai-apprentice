# 组件与视觉规范

## 视觉方向

“精密校核台”使用近白工作面、冷灰应用壳、石墨文字和 1px 分隔线。工程蓝只表示选择/参考关系，朱红表示修改，绿色表示保护，琥珀表示等待审核或阻断。圆角上限 6px，动效 140–180ms，无玻璃、光球、厚阴影或营销卡片。

## 共享 token

实现位于 `shared/tokens.css` 与 `design-tokens.json`：

- 字体：Noto Sans CJK SC / Source Han Sans SC / Microsoft YaHei UI 回退；ID、尺寸与定位器使用 Cascadia Mono / Consolas。
- 修改区：`--modify #d4473b`；保护区：`--protect #2d7a50`；参考关系：`--reference #1769a6`。
- 基础圆角：3/4/6px；所有交互反馈同时使用颜色、描边/形状、文字或 `aria-*`。

## 桌面信标

`.ai-beacon` 是 48×62px 固定入口，默认左下 14px。结构是矩形 `AI` 核心、状态灯、机械臂、圆形工业关节、立柱和矩形底座。它只做两件事：打开当前页面的上下文操作面；显示读取、监听、等待、成功、错误、离线状态。标签仅在 hover、键盘焦点或展开时出现，尽量减少对主工作区的遮挡。

状态灯映射：蓝=读取/操作已就绪，琥珀=等待审核，绿=成功或真实宿主回滚成功，红=错误，灰=离线。禁止添加脸、手势、气泡人格话术、口袋、铃铛或圆头角色轮廓。

## 上下文流程条

原生页面使用四步条：读取、意见、预览、审核。已完成步骤用绿色实心序号；当前步骤用蓝色实心序号和下边线。步骤条不是导航许可模型：宿主仍需根据后端状态决定哪些步骤可点击。

## 右键菜单

Word/Excel 菜单第一项为“交给 AI 学徒修改”，副文案说明读取原生选区与上下文。工程菜单第一项附对象 ID、类型和当前值。移动端由信标或宿主触控长按事件打开底部操作面。

## 检查器 / 操作面

桌面检查器按领域使用 382–400px，独立滚动，提交区固定在底部；移动端从底部升起，顶部保留关闭按钮。表单、事实列表、差异与状态直接使用分隔线组织，不做卡片套卡片。

## Canvas 分层

- 包装页：底图 + `referenceCanvas` + `maskCanvas`。
- Office 页：宿主内容 DOM + 只读关系 Canvas；实际写回使用原生 locator/range。
- 工程页：模型示意 Canvas + zone Canvas；实际写回使用对象 ID 和宿主桥动作。
