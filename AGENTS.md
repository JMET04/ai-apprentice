# AGENTS.md

## Project Vision

This project is called Transparent AI Apprentice MCP.

The goal is to build a universal teachable AI apprentice system. The product should allow humans to teach AI like teaching a student or training a new assistant. The AI should learn from visual demonstrations, natural language corrections, structured feedback, examples, and execution history.

The goal also includes a knowledge-augmented RAG research lane: ordinary users should be able to low-cost cultivate domain-specialized apprentices by providing manuals, standards, papers, software docs, teacher notes, prior corrections, examples, and log-format references. RAG is evidence only: it may retrieve source-backed context and draft disabled rules, but it must not enable rules, execute software, bypass teacher confirmation, write long-term memory, claim technology acceptance, or unlock packaging.

The core product principle is:

Human teaches -> AI executes -> human corrects -> system extracts rules -> AI improves -> execution becomes more controllable and transparent.

## Product Principles

- Do not design this as a generic chatbot.
- Do not design this as a cold workflow automation tool.
- Design it as an AI apprentice that can be taught, corrected, and trusted.
- Every important AI action should produce a visible, structured trace.
- Never expose private chain-of-thought. Show structured reasoning traces instead: steps, rules, confidence, validation results, and human review points.
- Human feedback should become reusable memory.
- The user should always understand why the AI produced a result.
- Retrieved knowledge should become provenance-carrying evidence packets, not hidden prompt magic or automatic authority.

## Architecture Principles

Keep these layers separate:

- UI layer
- API layer
- AI service layer
- workflow execution engine
- memory store
- trace store
- correction/rule extraction module
- knowledge source registry and retrieval evidence packet layer
- tool registry
- skill registry
- guardrail/policy layer

## Coding Rules

- Use TypeScript wherever possible.
- Keep components small and readable.
- Avoid hardcoded secrets.
- Use `.env.example`.
- Add clear comments around core learning, correction, and execution logic.
- Mock AI calls are acceptable in MVP, but they must be isolated behind service interfaces.
- Any mock should be easy to replace with a real model API later.
- Use seed data so the demo works immediately after setup.
- Prefer clear architecture over clever code.

## MVP Focus

Prioritize the full learning loop:

1. Create an apprentice.
2. Create a teachable task.
3. Build a visual workflow.
4. Execute the task.
5. Show structured execution trace.
6. Let the user correct the output.
7. Convert correction into reusable rules.
8. Apply the rules in the next run.

### 三维工程带教

`src/lib/spatial-teaching.ts` 提供完整的三维坐标带教模型：
- **代码优先输入**：老师用 JSON/DSL 直接输入 x/y/z 点列、容差、约束（轴向吸附/折线候选/自由意图），图片仅作可选参考
- **数学拟合多候选**：对粗略点列同时生成最小二乘线、轴向约束线和两段折线候选，公开残差、置信度和可选理由
- **圆弧意图候选**：在直线、轴向线和折线之外额外生成 x/z 平面的圆弧候选，公开圆心、半径、扫角、残差、方向容差和老师确认问题
- **平滑样条意图候选**：在人类粗线条不是严格圆弧或折线时，额外生成 cubic Bezier 平滑样条候选，公开控制柄、采样点、残差、方向容差和老师确认问题
- **曲面 patch 教学放大镜**：当老师可能在教局部曲面或高度面意图时，额外生成 `x/z -> y` 高度 patch 审查面，公开 patch 方程、梯度、四角、逐点高度残差、老师确认问题和下一步预测；该放大镜只用于审查，不启用规则、不解锁封装
- **曲面拟合意图候选**：老师需要把局部曲面/高度面作为真实意图时，系统可以把 patch 审查面转成 `surface_patch` 候选，接入候选矩阵、残差放大镜、方向容差、影响预演和二轮纠正；二轮纠正可以编辑纠正文本并选择纠正类型后本地即时重算，老师选中某个二轮走法后也只能生成 no-op follow-up plan、evidence path 和 verifier command，且 qualification API 会输出 `selectedSecondRoundPreview` 与对应 summary 计数供审查代理验证；仍只作老师审查，不自动保存、启用规则、验收技术或解锁封装
- **拟合候选对比矩阵**：所有候选必须汇总成老师可快速选择的对比表，展示建议审查顺序、残差排名、方向偏差排名、复杂度、老师选择提示和取舍说明
- **候选选择影响预演**：老师点选候选后，AI 必须提前展示 disabled 规则草稿、旧知识冲突边界、逐步下一手预测、老师纠正点和记忆策略，不得静默启用规则
- **批量示教数学建模**：多次示教样本汇总为共识模型，公开稳定锚点、y/z 漂移范围、跨度变化和离群保护
- **位置参数学习报告**：批量示教进一步拆解为起点 y、终点 y、起点 z、终点 z、跨度、方向漂移、推荐容差和离群策略，公开均值、范围、标准差和老师确认问题
- **曲面 patch 批量稳定性报告**：批量示教进一步拆解 surface patch 的 x/z 梯度共识和高度残差包络，公开稳定样本数、离群保护、老师确认问题和封装锁，不自动保存或启用规则
- **曲面批量老师选择回放**：老师选择采用稳定共识、收紧残差容差或隔离离群样本时，系统提前展示 disabled 规则草稿、旧知识冲突边界和下一步预测；仍只作审查，不保存验收、不解锁封装
- **拟合误差教学放大镜**：在 SVG 三维画布上绘制每个候选的残差向量，橙红色标记超容差点，淡灰显示未超容差点
- **构造预测和纠正**：每个候选生成锚点、方向向量和 3 步构造预测，老师可逐步纠正
